# 把「真·贵州」搬进游戏：实景高斯泼溅（3DGS）落地管线

> 目标：让 `guizhou/p4-splat.html` 里 5 个地区的背景，从"通用真实场景"换成**真正的黄果树 / 梵净山 / 小七孔 / 千户苗寨**。
> 结论先行：**网上没有现成的真·贵州泼溅可下载**（我搜了 HF / Sketchfab / Luma / Polycam / 知网论文，均无）。所以 C 只有一条路——**自己（或找人）实地拍 → 转成泼溅 → 塞进本系统**。好消息：本系统已经为这一步铺好路（见 §4）。

## 1. 一句话路线

**用手机 App「Scaniverse」实地扫 → 导出 `.spz` → 在游戏面板点「上传 .spz/.ply」→ 走进去。**

为什么是这条：Scaniverse、`.spz` 格式、本游戏用的 Spark 渲染器**都是 Niantic 同一家做的**，格式天生兼容；Scaniverse 免费、手机本地处理（不上传云端）、自带重力对齐（拍出来不会歪）。

## 3. 各地点拍摄要点 & 现实预期

3DGS 的软肋：**动的水、纯天空、超大远景、镜面**会糊或飘。按"可拍性"排序：

| 地点 | 怎么拍 | 难度 | 现实预期 |
|---|---|---|---|
| **荔波小七孔·古桥/卧龙潭** | 手持绕桥/沿岸走 30–60s，多角度 | ★★ 易 | 桥体、林子、岸边极好；**流动的水面会糊**，静潭还行 |
| **千户苗寨·吊脚楼/观景台** | 观景台俯瞰 + 走巷子分段扫，每栋楼绕拍 | ★★★ 中 | 单栋吊脚楼/一段巷子很好；**整片万家灯火**要无人机夜拍，难 |
| **梵净山·红云金顶/蘑菇石** | 无人机绕飞金顶；蘑菇石可手持环绕 | ★★★★ 难 | 蘑菇石、寺这种"物体"好；**云海是动的→会糊**，当氛围尚可 |
| **黄果树·瀑布** | 无人机绕飞 + 岸边多机位；瀑布本体当"动态纹理" | ★★★★★ 最难 | 崖壁/水帘洞/植被好；**飞流的水 3DGS 基本搞不定**，会成一片雾。可拍"水帘洞内+崖壁"，水当背景 |

> 诚实说：黄果树瀑布的"飞瀑"是 3DGS 最难的题（高速运动 + 水花）。真要这个镜头，更稳的是**实拍视频**直接当背景（面板已支持「上传视频」），泼溅留给能站定环视的实景（洞、桥、寨、石、寺）。

**通用拍摄守则**（决定成败）：
1. **绕着主体走一整圈**，别站着转——要"位移"视差，相机原地转出不来 3D。
2. 同一物体**从高/中/低三个高度**各绕一圈最佳。
3. 光线均匀（阴天/清晨最好），避免大逆光、过曝。
4. 30–90 秒 / 200–500 张的量级；太多反而慢、易飘。
5. 别拍纯天空、大镜面、大片动水当主体。

## 4. 怎么塞进游戏（系统已就绪）

### 方式 A：本地上传（最快，验证用）
1. 打开 `guizhou/p4-splat.html` → 进入 → 右下「面板」。
2. **实景 3DGS → ⬆ 上传 .spz/.ply**，选你导出的文件。
3. 本地直接加载（不传服务器）。系统自动取景；若歪了点 **「校平」**（PCA 找地平面摆正）或 **「翻转」**（上下翻 180°）。

> 上传是临时预览（刷新即失），用来快速看效果。要固定进某个地区，用方式 B。

### 方式 B：固定进 5 个地区（正式）
1. 把 `.spz` 放到能访问的地址：
   - 放进项目 `guizhou/assets/splats/你的文件.spz`，用本地服务器；或
   - 传到带 CORS 的图床/HF（`huggingface.co/.../resolve/main/xxx.spz` 自带 CORS）。
2. 编辑 `p4-splat.html` 顶部的 `SPLATS` 数组，把对应地区那行的 `url` 换成你的：
   ```js
   { name:'黄果树·水帘洞', mb:60, url:'assets/splats/huangguoshu.spz',
     rot:[Math.PI,0,0], size:34, anchor:[0,1.5,-3] },           // Scaniverse 的 .spz 用这套
   // 若是 Postshot/COLMAP 的 .ply、朝向乱：改成自动校平
   // { ..., url:'...huangguoshu.ply', rot:[0,0,0], autolevel:true, size:32, anchor:[0,2,-3] },
   ```
   - **Scaniverse 的 `.spz`**（已重力对齐）→ 用 `rot:[Math.PI,0,0]` + `size/anchor`（同 valley 那类）。
   - **Postshot/COLMAP/Luma 的 `.ply`**（朝向任意）→ 用 `autolevel:true`（PCA 自动摆正）。
   - 取景微调：进游戏开 DevTools 控制台，实时调 `__reframe({size:30, anchor:[0,2,-3], rot:[0,0,0]})`，满意后把值写回数组。

### 取景参数速查
- `scale`+`pos`：**显式**取景（最稳，调好就钉死）。
- `size`：场景框进世界的目标直径（越大越近）。`anchor`：场景中心锚到的世界坐标（y 越小越贴地）。
- `autolevel:true`：开 PCA 自动校平（朝向乱的照片重建用）。
- `rot:[x,y,z]`：欧拉弧度，叠加在自动校平之上的手动修正（翻转/转向）。

## 5. 路线图建议

1. **先打样**：挑最容易的**荔波小七孔古桥**，Scaniverse 手持绕拍 → 导 `.spz` → 上传看效果。一次成功就建立信心。
2. **再扩 2–3 个**站得定的点（寨门/吊脚楼/蘑菇石/水帘洞），逐个换进 `SPLATS`。
3. **大景/动水**（瀑布全景、云海、万家灯火）：用**实拍视频**当背景（面板「上传视频」）兜底，或后期无人机 + Postshot 再补。
4. 全部到位后，5 个地区就真的是"用游戏的方式打开贵州"了。

## 6. 诚实的边界

- 3DGS 拍**能站定环视的实景**（桥/洞/楼/石/寺/巷）效果惊艳；拍**动水/云海/纯远景**会糊，别强求。
- 大场景质量上限在"拍摄覆盖度"，不在工具——**绕圈走位**比换工具重要得多。
- 真贵州落地的瓶颈是**实地拍摄**（要去现场），这一步我替代不了；但拍回来之后的"转泼溅→进游戏"已全部铺好。

# 用网页做精美动作游戏：技术调研报告（2026-06）

> 由多智能体并行检索 + 逐条事实核查生成（7 维度 × 研究/核查 + 完整性总评，共 15 个智能体）。
> 配套有三款可玩 Demo（见 `index.html`），打开浏览器即可亲手体验难度梯度。

---

## 0. 一句话结论

**难度不取决于"画面有多炫"，而取决于一条铁律：难度 ≈ 延迟敏感度 × 状态同步精度。**
按两个轴划分：

| | **单机** | **联机** |
|---|---|---|
| **2D** | ⭐⭐ 甜点区，个人 1–3 个月可出门户级精品 | ⭐⭐⭐⭐ 工作量是单机数倍 |
| **3D** | ⭐⭐⭐ 管线工程 + 性能预算 | ⭐⭐⭐⭐⭐ 天花板（尤其格斗 rollback / 竞技 FPS） |

- **2D 单机 = 做对就好看**：技术栈极成熟，靠"统一低分辨率 + 限定调色板 + 强后处理 + 到位的 juice"就能用廉价/CC0 资产撑出"精美感"。这是个人开发者最该先吃下的领域。
- **3D 单机 = 中高**：难在美术/绑骨管线、物理、相机；"Unity/Godot 一键导 Web"有被低估的包体/内存/线程工程税。
- **任何联机 = 真正的天花板**：要叠加权威服务器、客户端预测、回滚/插值、延迟补偿；传输层必须用 WebRTC/WebTransport 而非 WebSocket。

## 1. 渲染底层：Canvas2D / WebGL2 / WebGPU 怎么选

| 路径 | 定位 | 上手 | 适合 | 拐点 |
|---|---|---|---|---|
| **Canvas 2D** | CPU 立即模式 | 最低 | 中小规模 2D（休闲、解谜、简单平台跳跃） | 对象一多 CPU 撞墙（基准：5 万点 Canvas2D 22 FPS vs WebGL 58 FPS） |
| **WebGL / WebGL2** | GPU 加速，当前事实标准 | 中（实战都用引擎封装） | 海量精灵/弹幕/粒子、着色器特效、任何 3D、稳定 60FPS 的精美 2D | 绝大多数精美动作游戏的**默认起点** |
| **WebGPU** | 新一代 GPU API，含 compute shader | 高（用引擎可降门槛） | 大规模 GPU 粒子/流体、GPU 计算、近桌面级画质 | 画质/规模天花板的进阶项 |

**2025–2026 关键变化**：WebGPU 已在四大浏览器默认开启（Chrome/Edge 2023 起、Safari 26 于 2025-09、Firefox 141/145）。但仍碎片化（Firefox 的 Linux/Android、旧 OS、低端移动 GPU），**生产姿态 = WebGPU 优先 + WebGL2 自动回退**。

**运行时三件套（动作游戏的地基）**：
1. **固定时间步长 + 累加器 + 渲染插值** —— 否则 120Hz 屏幕物理会快一倍、且会穿墙。配单帧步数封顶防"死亡螺旋"。
2. **对象池 + 热路径零分配** —— V8 的 GC 是卡顿主因；子弹/粒子/向量都要池化（社区实测降 GC 压力约 90%）。
3. **（可选）OffscreenCanvas + Web Worker** —— 把渲染移出主线程，输入响应更稳。

## 2. 引擎与框架（按目标选）

| 名称 | 2D/3D | 类型 | 开源 | 一句话 |
|---|---|---|---|---|
| **Phaser 4** | 2D | 框架（batteries-included） | ✅ | 2D 默认首选，2026-04-10 正式版，新 WebGL 渲染器 + 一行启用动态光照/自阴影 |
| **PixiJS v8** | 2D | 纯渲染库 | ✅ | 最快 2D 渲染，弹幕/粒子重的项目；需自己搭玩法层 |
| **Three.js** | 3D | 库 | ✅ | 3D web 事实标准（~11.2 万 stars、周下载 ~540 万），灵活但要自己接物理 |
| **Babylon.js 8** | 3D | 完整引擎 | ✅ | 功能最全，WebGL/WebGPU 双轨，内置物理/动画/XR，文档友好 |
| **PlayCanvas** | 3D | 引擎+编辑器 | ✅（含编辑器，2025-07 起 MIT） | 协作 + 移动优先，包体最小最快，runtime 常 <200KB |
| **Cocos Creator 3.8** | 2D/轻3D | 引擎+编辑器 | ✅ | 微信/抖音小游戏生态首选，分包/包体优化成熟 |
| **Construct 3 / GDevelop** | 2D | 无代码 | C3 闭源订阅 / GDevelop 开源 | 最易上手，事件表可视化逻辑 |
| **KAPLAY**（原 Kaboom） | 2D | 轻量库 | ✅ | game jam / 教学最快原型 |
| **Defold** | 2D/轻3D | 引擎 | ✅ | 业界最小包体之一（HTML5 ~1MB gzip） |
| **Unity WebGL** | 全 | 商业引擎导出 | ❌ | 功能最强但 web 是次要目标：空项目包体可达 10MB+、**无 C# 多线程**、内存易失败 |
| **Godot 4 Web** | 全 | 开源引擎导出 | ✅ | 4.3+ 支持**单线程导出**（免 SharedArrayBuffer），根治 itch.io 部署坑 |
| **Bevy (Rust+wasm)** | 全 | ECS 引擎 | ✅ | 性能强但包体最大、需 Rust |

**决策建议**：2D 动作 → Phaser 4（或 PixiJS + 物理库）；3D 动作 → Three.js + Rapier 或 Babylon.js/PlayCanvas；要上中国小游戏 → Cocos Creator；想要"现成引擎最省工" → Unity WebGL（代价是包体/首屏）。

## 3. 动作子类型难度排行榜（网页实现）

| 子类型 | 难度 | 为什么 |
|---|---|---|
| 无尽跑酷 Endless Runner | **入门** | 机制最简单，程序化分段 + 难度曲线 + 对象回收；最适合触屏 |
| 平台跳跃 Platformer | **入门** | 轮子成熟（AABB 碰撞、变高跳、coyote time、jump buffer）；顶级手感单机可达（Celeste 级） |
| 双摇杆 / 刷子 | **中等** | 大量实体 → 对象池 + 空间划分（网格/四叉树）+ 双轴输入解耦 |
| 弹幕 Bullet Hell | **中等** | 瓶颈是性能：去掉物理碰撞器改自写点判定 → 可上 10000+ 子弹不掉帧 |
| 横版清关 Beat-em-up | **中等** | 多一个伪 3D 的 Z 轴纵深判定 + 敌人 AI 编组（攻击排队） |
| 节奏 Rhythm | **中等** | 逻辑不难，难在**延迟校准**（音/视/输入三类延迟逐设备不同，蓝牙可加 100–300ms） |
| 3D 赛车 Racing | **高** | 车辆物理（悬挂 + 四轮 raycast）、相机、LOD、移动端多边形预算 |
| 类魂 / Character Action | **高** | "动画优先"承诺、逐帧 hitbox 标签、取消窗口、庞大招式树、风评系统 |
| 第一人称射击 FPS（单机） | **高** | Pointer Lock 无界视角、射线命中、武器手感；门槛高于 2D |
| 实时同屏合作/对战（联机化） | **很高** | 权威服务器 + 预测 + 重对账 + 插值，外加 NAT 穿透/反作弊 |
| 竞技格斗 rollback / 竞技 FPS 帧级公平 | **很高（天花板）** | 确定性模拟（常需放弃 float 改定点 Q16.16）+ 快速 save/load + 延迟补偿 + 反作弊 |

## 4. "精美 + 打击感"从哪来（juice 是最便宜的高级感）

**画面是否"好"，很大程度是反馈是否到位。这部分几乎零美术成本，却最显精致。**

- **hit-stop / 顿帧**：命中瞬间冻结攻防双方 2~8 帧（越重越久；《Final Fight》固定 6 帧）。不是 setTimeout 暂停，而是在固定步长里跳过若干 tick。
- **屏幕震动**：用 `trauma∈[0,1]`，位移 = trauma² 或 trauma³ + Perlin 噪声 + 线性衰减（Squirrel Eiserloh, GDC 2016），不要纯 random。
- **粒子 / 闪白 / 拖尾**：命中把被击者整体染白 1~2 帧；粒子走 GPU 实例化 + 对象池。
- **相机**：lerp 平滑跟随 + lookahead（按速度前移镜头）+ dead zone（死区）+ SmoothDamp。
- **音画同步**：必须 Web Audio（预解码 AudioBuffer + `ctx.currentTime` 精确排程），人耳容忍 ~20ms；`<audio>` 标签延迟太高不可用。
- **squash & stretch**：跳起纵向拉伸、落地横向压扁。
- **手感容错**：coyote time（离台后仍可跳几帧）+ jump buffering（落地前提前按跳自动起跳）—— Celeste 的精髓。

**美术管线（低成本做精美）**：
- 像素：**Aseprite**（$19.99）/ 免费用 Pixelorama；关卡用 **Tiled / LDtk**；图集用 free-tex-packer 降 draw call。
- 骨骼动画：**Spine**（商业标准，$69/$369）/ **DragonBones**（免费但近年停更）。
- 后处理：PixiJS Filters（AdvancedBloom / Glow / CRT）、Phaser 4 一行动态光照 + 法线贴图。
- 3D：**Blender → glTF/GLB + Draco（几何降 80–95%）+ KTX2/Basis（显存省 ~75%）**，动画用 Mixamo 自动绑骨。
- 音频：jsfxr/BFXR 生成 + Howler.js（~7KB）播放。
- 免费资源：**Kenney（CC0，6 万+资产）**、OpenGameArt、itch.io。
- **底层规律**：统一低分辨率（如内部 320×180 整数放大）+ 限定调色板（16/32 色）+ nearest 采样 —— 风格统一比单张图精细更显高级。
- AI 工作流（2026 常态）：Retro Diffusion / PixelLab 批量生成候选 → Aseprite 精修。

## 5. 联机：真正的难度天花板

| 技术 | 作用 |
|---|---|
| **传输层** | WebSocket（TCP，队头阻塞）**不适合时敏游戏**；用 WebRTC DataChannel 或 WebTransport（QUIC/UDP，2026 起全浏览器覆盖） |
| **权威服务器** | 状态只由服务器决定，客户端发输入，防作弊 |
| **客户端预测 + 重对账** | 本地立即模拟自己 + 服务器回包后重放未确认输入纠错（消除一个 RTT 的延迟感） |
| **实体插值** | 远端渲染在"过去"（缓冲历史快照间插值），消除瞬移 |
| **延迟补偿** | 服务器回溯到射击者看到的那一刻判定命中（FPS） |
| **rollback netcode** | 格斗金标准（GGPO 范式）：预测远端输入立即模拟，错了就回滚重放；要求确定性 + 快速 save/load |

**确定性的坑**：浮点在不同 CPU/浏览器结果不同 → 跨端 desync。锁步/回滚需放弃 float 改**定点数 Q16.16**，或用确定性物理（`@dimforge/rapier3d-deterministic`）。

**务实建议**：先把单机手感做到极致；联机能用"状态同步 + 客户端预测 + 实体插值"（允许 float）就别上锁步/回滚，除非是格斗/高精度对战刚需。入门读 **Gabriel Gambetta《Fast-Paced Multiplayer》**（四篇 + <500 行 JS demo）。

## 7. 发行与变现（2025–2026）

- **头部门户**：**Poki**（~1 亿月活，自带流量拿 100% / Poki 流量 50/50，编辑精选制）；**CrazyGames**（~3500 万月活，内购回本后 ~70%，满足 4 条件 +50% 加成）。
- **聚合器**（最快变现入口）：GameMonetize / GameDistribution / GamePix，非独家铺量，纯开发者基础分成约 45%。
- **上 Steam**：NW.js（最省心，Steam 上最常见 JS 方案）/ Electron（最成熟）/ Tauri（最小但需 Rust）。
- **移动端**：Capacitor（优于老旧 Cordova，重图形游戏务必真机实测）。
- **新战场**：Discord Activities（~2 亿月活、月游玩 15 亿+ 小时）、Telegram Mini Games、**中国微信/抖音小游戏（2025 约 535 亿元，+34%）**。
- **开发经济学**：典型网页游戏 1–3 个月、成本 <1 万美元；门户对个人开发者的分成结构远优于移动商店（15–30% 抽成）。建议组合"门户广告/内购 + 非独家授权 + 付费桌面版/众筹"。

## 8. 推荐学习路径

| 阶段 | 时长 | 重点 |
|---|---|---|
| **0 打地基** | 1–2 周 | 纯 JS+Canvas2D 手写：rAF + 固定步长 + 累加器 + 渲染插值 + 对象池意识。先理解为什么裸写会撞墙 |
| **1 2D 单机精品** | 4–8 周 | Phaser 4 做完整平台跳跃/双摇杆；学透碰撞、coyote/buffer、Tiled、图集；练"精美感杠杆"+ 完整 juice + Web Audio。**产出即门户级精品** |
| **2 子类型专精** | 4–6 周 | 选一类深耕（弹幕自写判定 / 类魂动画事件帧 / 赛车物理）；串起骨骼动画 + 法线光照 + 后处理管线 |
| **3 发行与多端** | 2–3 周 | 聚合器铺量 → 争 Poki/CrazyGames 精选；NW.js/Electron 上 Steam；Capacitor 封装；老 iPhone+低端安卓真机实测 |
| **4 进入 3D（可选）** | 6–10 周 | Three.js / Babylon.js；建 Blender→glTF+Draco+KTX2 管线；Mixamo 绑骨；Rapier 物理；WebGPU 优先 + WebGL2 回退 |
| **5 联机（高阶，仅必要时）** | 8 周+ | 先做"宽松实时"（权威服务器+预测+插值，允许 float）；FPS 加延迟补偿；只有竞技格斗/高精度对战才上 rollback/锁步（届时解决浮点确定性）。反作弊当一等公民 |

## 9. 给你的建议

1. **从 2D 单机开始**，目标是 1 个月内做出一款手感扎实、有 juice 的小游戏 —— 这是性价比最高、最能建立信心的路径。
2. **手感 > 画面**：先把"输入→动作"延迟压到最低，再加特效。过度 juice 会掩盖核心玩法问题。
3. **联机能不上就别上**：它是单机难度的数倍。真要联机，先用宽松实时方案。
4. **3D 的瓶颈通常是美术不是代码**：单人很难同时兼顾程序+建模+绑骨+动画，优先买现成资产/CC0。
5. **真机测试是生命线**：尤其老款 iPhone（Safari/Metal 内存限制）和会热降频的低端安卓。
